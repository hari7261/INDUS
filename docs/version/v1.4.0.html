Overview Commands Versions v1.4.0
Released March 8, 2026

v1.4.0 turns INDUS into a fully native command platform.

This release introduces 58 native commands, a central registry at core/commands/registry.json, lazy-loaded modules, command caching, safe error wrappers, versioned docs, and AI-readable metadata.

58 commands 10 modules registry version 1.4.0 AI command index

Headline changes

  • Independent ind ... namespace
  • Cache-aware command engine
  • Cross-platform console hooks
  • Versioned docs UI with SEO metadata

New command families

  • Core: scan, doctor, status, docs, about, version
  • System: stats, info, clean, doctor, watch
  • Project: create, init, list, clean, build, run
  • Environment, filesystem, network, developer, package, terminal, and workspace modules

Engine architecture

  • Registry lookups before execution
  • Module factories with lazy load on first hit
  • Async execution wrapper and panic recovery
  • Short-lived result caching for read commands

Migration guide

  • Replace bare commands with the ind prefix.
  • Use ind proj ... for project lifecycle work.
  • Use ind net fetch for native HTTP requests.
  • Open commands.html for the full searchable reference.

Verification targets

  • Local commands target under 50ms.
  • Complex commands target under 150ms.
  • Read commands opt into short cache TTLs.
  • Invalid input returns structured IND error codes.
